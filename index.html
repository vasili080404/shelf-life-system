The Problem

Manual shelf-life control at gas station #16 led to errors, lost revenue and full dependence on human factor. Duplicate data between 1C and the Operator ARM, no automatic block of expired sales, no FEFO logic in the ordering process.

The Solution

A prototype system that implements FEFO (First Expired — First Out), automatic batch status detection (Normal / Critical / Expired), hard block of expired sales, intelligent auto-ordering based on average sales and planogram, and supplier return with an act of disposal.

Implemented Features

⚡

Automatic status calculation

Normal · Critical (1–3 days) · Expired. Color-coded indicators in the operator UI, computed automatically from the expiration date.

🧮

Auto discount calculation

Discount percentage for critical batches is computed automatically from configurable rules.

🔍

FEFO batch picking

When a product is scanned, the batch with the nearest expiry is selected automatically. Selling expired goods is hard-blocked.

🛒

Intelligent auto-order

Takes into account 14-day average sales, current stock, planogram and criticality. Recommendation is generated automatically.

🗑️

Expired write-off

Mass write-off of expired batches with auto-generated act and full logging into the ActionLog table.

↩️

Supplier return

Automatic selection of batches with critical expiry (≤ 14 days), return act generated with reason.

📋

Planogram

Digital control of product placement on shelves with min/max stock thresholds per slot.

📊

Reports

Catalog, sales history, 7-day losses, write-off history, my orders, order details, auto-order proposal. Export to TXT and Excel.

🔐

Full audit trail

Every user action is logged into the ActionLog table: who, when, what changed.

Architecture

Three-tier modular architecture. Modules communicate only through the shared SQLite database, which keeps coupling minimal and the system easy to test.

Presentation Layer ui/
app.py operator_tab.py senior_tab.py dialogs.py notify.py
▼
Business Logic Layer modules/

▼
Data Access Layer modules/database.py
Database class · 10 SQLite tables · context manager · rows_to_dicts · CRUD methods

Technology Stack

Language: Python 3.11
GUI: CustomTkinter (modern wrapper over Tkinter)
Database: SQLite (10 tables, WAL mode)
Report export: openpyxl (Excel), plain text (TXT)

Testing: manual smoke tests + integration test_modules.py

How to Run Locally

# 1. Clone the repository
git clone https://github.com/vasili080404/shelf-life-system.git
cd shelf-life-system

# 2. Install dependencies
pip install -r requirements.txt

# 3. Move into the source directory and initialize the database
cd shelf_life_system
python init_db.py

# 4. Launch the application
python app.py

# Optional: screenshot mode for the diploma
python app.py --tab=senior --screenshots=D:\shots --delay=2000

# CLI modes: --tab=senior, --open=my_orders,
# --open=writeoff_history, --open=action_log, --open=planogram
