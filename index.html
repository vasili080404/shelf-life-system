Проблема

На АЗС №16 ручной контроль сроков годности приводил к ошибкам, потерям выручки и зависимости от человеческого фактора. Дублирование данных между 1С и ARM Оператор, отсутствие автоблокировки просроченных продаж, отсутствие FEFO-логики при формировании заказов.

Решение

Прототип системы, реализующий принцип FEFO (First Expired — First Out), автоматическое определение статуса партий (Normal / Critical / Expired), блокировку продаж просроченного товара, интеллектуальный авто-заказ на основе средних продаж и планограммы, возврат поставщику и списание с формированием акта.

Реализованные функции

⚡

Автоматический расчёт статуса

Normal · Critical (1–3 дня) · Expired. Цветовая индикация в интерфейсе оператора, автоматическое определение по дате истечения.

🧮

Авто-расчёт скидок

Процент скидки для критических партий рассчитывается автоматически на основе настраиваемых правил.

🔍

FEFO-выбор партии

При сканировании товара автоматически выбирается партия с ближайшим сроком истечения. Продажа просрочки заблокирована.

🛒

Интеллектуальный авто-заказ

Учёт средних продаж за 14 дней, текущих остатков, планограммы, критичности сроков. Рекомендация формируется автоматически.

🗑️

Списание просрочки

Массовое списание просроченных партий с автоматическим формированием акта и логированием в ActionLog.

↩️

Возврат поставщику

Автоматический выбор партий с критическим сроком (≤14 дней), формирование акта возврата с указанием причины.

📋

Планограмма

Цифровое управление размещением товаров на полках с указанием минимальных и максимальных остатков.

📊

Отчёты

Каталог, история продаж, потери за 7 дней, история списаний, мои заказы, детали заказа, предложение авто-заказа. Экспорт TXT и Excel.

🔐

Полный аудит

Каждое действие пользователя логируется в таблицу ActionLog: кто, когда, что изменил.

Архитектура

Трёхуровневая модульная архитектура согласно BPMN-диаграммам дипломной работы. Модули связаны только через общую БД, что обеспечивает минимальное сопряжение и простоту тестирования.

Технологический стек

Язык: Python 3.11
GUI: CustomTkinter (модерн-обёртка над Tkinter)
БД: SQLite (10 таблиц, WAL-режим)
Экспорт отчётов: openpyxl (Excel), plain text (TXT)
Звук: winsound (системные уведомления)
Тестирование: ручные smoke-тесты + интеграционный test_modules.py
Объём кода: 3 320 строк Python в 17 файлах
Методология требований: FURPS+

Как запустить локально

# 1. Клонировать репозиторий
git clone https://github.com/vasili080404/shelf-life-system.git
cd shelf-life-system

# 2. Установить зависимости
pip install customtkinter openpyxl

# 3. Инициализировать БД
python init_db.py

# 4. Запустить приложение
python app.py

# Опционально: режим скриншотов для диплома
python app.py --tab=senior --screenshots=D:\shots --delay=2000

# CLI-режимы: --tab=senior, --open=my_orders,
# --open=writeoff_history, --open=action_log, --open=planogram
